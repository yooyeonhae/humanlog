# 인간로그 개발 프롬프트 (AI 코딩 도구 전달용)

> 이 문서를 Claude Code 등 AI 코딩 도구에 그대로 붙여넣어 개발을 시작할 수 있습니다.
> PRD → UI 명세/사이트맵 → 와이어프레임 단계를 거쳐 정리된 최종 개발 스펙입니다.

---

## 0. 한 문장 요약

"사람을 분류하지 않는다. 사람을 이해하게 만든다."는 철학의 자기관찰 다이어리 웹앱 **인간로그**를,
**HTML5 + CSS3 + Vanilla JavaScript**만으로 구현하고 **Vercel**에 무료 배포하며,
사용자 기록은 **브라우저 로컬 저장(localStorage) + 엑셀(.xlsx) 백업**으로 관리한다.

---

## 1. 절대 지켜야 할 톤 & 금지사항 (모든 화면 공통)

- MBTI 스타일 유형 라벨("당신은 ENFP형") 절대 사용 금지
- "당신은 OO한 사람이다" 식 단정 표현 금지 → 항상 **"~일 수 있어요"** 잠정 어조
- 외모 평가, 우열 비교, "누가 누구와 더 잘 맞는다" 식 궁합 표현 금지
- 친구 연결·알림 기능은 전부 **선택 사항**이며, 이 기능들을 전혀 쓰지 않아도
  오늘의 질문 → AI 관찰 → 기록 보기 → 백업까지 모든 핵심 경험이 완결되어야 함
- 위 원칙은 UI 카피뿐 아니라 데이터 모델과 빈 상태(Empty State) 문구에도 동일하게 적용

## 2. 기술 스택 & 제약사항

| 구분 | 내용 |
|---|---|
| 마크업/스타일 | HTML5, CSS3 (프레임워크·전처리기 없이 순수 CSS) |
| 로직 | Vanilla JavaScript (ES6+, 번들러/프레임워크 없음) |
| 배포 | Vercel 무료 플랜 — 정적 사이트로 배포 (별도 백엔드 서버 없음) |
| 데이터 저장 | 1차: `localStorage` / 백업: `.xlsx` 파일 (SheetJS, CDN 스크립트로 로드) |
| 라우팅 | 별도 서버 라우팅 없이 **해시 기반 클라이언트 사이드 라우터**로 구현 (`location.hash` 감지 → 화면 전환) |
| 외부 리소스 | Google Fonts(Noto Sans KR, Gowun Batang), SheetJS(xlsx.full.min.js) — 둘 다 CDN에서 로드 |

## 3. 폴더/파일 구조

```
human-log/
├── index.html                # 단일 진입점 (SPA 셸)
├── vercel.json                # 정적 배포 설정 (필요 시 헤더/캐시만 지정)
├── css/
│   ├── reset.css              # 최소한의 리셋
│   ├── tokens.css             # 디자인 토큰(색상/폰트/라운드 변수)
│   └── styles.css             # 전체 레이아웃 및 컴포넌트 스타일
├── js/
│   ├── app.js                 # 앱 초기화, 라우터 등록
│   ├── router.js              # 해시 기반 라우터
│   ├── data/
│   │   ├── questions.js       # 질문·선택지 mock 데이터
│   │   └── insights.js        # AI 관찰 mock 문장 pool + 생성 함수
│   ├── storage.js             # localStorage 읽기/쓰기 유틸
│   ├── xlsx.js                # SheetJS 이용한 내보내기/가져오기 로직
│   └── views/
│       ├── onboarding.js
│       ├── home.js
│       ├── log.js
│       ├── logDetail.js
│       ├── backup.js
│       ├── connect.js
│       ├── connectCompare.js
│       ├── settings.js
│       └── notificationSettings.js
└── assets/
    └── icons/                 # 필요한 경우 svg 아이콘
```

- 각 `views/*.js`는 `render(container, params)` 함수를 export하고, `router.js`가 해시에 맞는 뷰를 찾아 호출한다.
- 공통 레이아웃(헤더 + 하단 네비게이션)은 `app.js`에서 한 번만 그리고, 내부 콘텐츠 영역만 뷰가 교체한다.

  --color-navy: #7C89B3;

  --radius-card: 24px;
  --radius-pill: 999px;
  --shadow-soft: 0 8px 30px -12px rgba(46, 44, 52, 0.10);

}

@media (prefers-color-scheme: dark) {

    --color-bg: var(--color-bg-dark);

- 모든 색상은 반드시 위 변수를 통해서만 사용하고, 하드코딩된 hex 값을 CSS에 직접 쓰지 않는다.

## 5. 데이터 모델 (localStorage)

```js
// key: "humanlog_responses"
// value: JSON.stringify(QuestionResponse[])
{
  date: "2026-07-06",        // YYYY-MM-DD, 하루 1건 원칙
  questionId: "q2",
  questionText: "새로운 사람을 만나면 보통 어떻게 행동하나요?",
  optionId: "think",
  optionLabel: "조금 생각한다",
  observation: "확신이 서기 전까지 한 발 물러서 관찰하는 편일 수 있어요."
}

// key: "humanlog_onboarded" → "true" | null
// key: "humanlog_theme" → "light" | "dark" | null (null이면 시스템 설정 따름)
// key: "humanlog_notification_settings" → { enabled: boolean, time: "20:00" }
// key: "humanlog_last_export_at" → ISO 날짜 문자열
```

- 절대 "성격 유형" 필드를 저장하지 않는다. 저장 단위는 항상 "그날의 개별 기록"이다.
- 하루 1회 원칙: 오늘 날짜의 기록이 이미 있으면 홈 화면은 그 기록을 그대로 복원해서 보여주고, 새로 답변받지 않는다.

## 6. 엑셀 내보내기 / 가져오기 (`js/xlsx.js`)

- 라이브러리: `<script src="https://cdn.jsdelivr.net/npm/xlsx/dist/xlsx.full.min.js"></script>`
- **내보내기**: `humanlog_responses` 배열을 (날짜, 질문, 선택, AI 관찰) 4개 열로 매핑 → `XLSX.utils.json_to_sheet` → `human-log-backup.xlsx` 다운로드
- **가져오기**: 업로드된 파일을 `XLSX.read`로 파싱 → 날짜 기준으로 기존 기록과 병합(같은 날짜면 가져온 데이터로 덮어쓰기) → `localStorage` 갱신 → `#/log`로 이동
- 내보내기 성공 시 `humanlog_last_export_at`을 현재 시각으로 갱신

## 7. 화면별 구현 스펙

> 각 화면은 3단계에서 만든 UI 명세의 "기타 UI 설계 정보"를 그대로 구현 기준으로 따른다.
> 아래는 화면별 핵심 동작만 요약한다.

### 7.1 온보딩 — `#/onboarding` (PRD 기능: 해당 없음, 철학 소개)
- 3장 슬라이드(가로 스와이프 또는 "다음" 버튼), 점 인디케이터 표시
- 마지막 슬라이드 버튼 텍스트만 "시작하기"로 변경 → 클릭 시 `humanlog_onboarded = "true"` 저장 후 `#/`로 이동
- `humanlog_onboarded`가 이미 있으면 앱 최초 로드 시 이 화면을 건너뛰고 `#/`로 바로 진입

### 7.2 홈 — `#/` (PRD 기능: ①②)
- `questions.js`에서 랜덤 질문 선택(단, 오늘 이미 답한 기록이 있으면 그 질문을 그대로 표시)
- 4지선다 버튼 클릭 → 선택 상태 표시(체크 아이콘) → `insights.js`의 `getInsight(questionId, optionId)`로 관찰 문장 생성(0.4~0.6초 지연 후 표시하는 스켈레톤 UI 필수)
- 답변 즉시 `humanlog_responses`에 저장(당일 날짜 기준 upsert)
- "다음 질문" 버튼은 답변 완료 전까지 `disabled`
- 하단에 "이 관찰은 당신을 규정하지 않아요" 문구 고정 표시

### 7.3 기록 아카이브 — `#/log` (PRD 기능: ③)
- `humanlog_responses`를 최신순 정렬, `YYYY년 M월` 단위로 그룹핑해 렌더링
- 각 아이템 클릭 시 `#/log/{date}`로 이동
- 기록이 없으면 Empty State: "아직 쌓인 기록이 없어요" + 홈으로 이동 버튼
- 우상단 아이콘 클릭 시 `#/backup`으로 이동

### 7.4 기록 상세 — `#/log/:date` (PRD 기능: ③)
- 해시 파라미터의 날짜로 `humanlog_responses`에서 해당 기록을 찾아 전체 내용(질문 전문, 선택, 관찰 전문) 표시
- 이전/다음 날짜 기록으로 이동하는 버튼(있는 경우만 활성화)
- 존재하지 않는 날짜로 접근 시 `#/log`로 리다이렉트

### 7.5 데이터 백업 — `#/backup` (PRD 기능: ④)
- "내보내기" 버튼 → 6절의 내보내기 로직 실행
- 드래그앤드롭 영역 + "파일 선택하기" 버튼 → 6절의 가져오기 로직 실행
- `humanlog_last_export_at` 값을 상단에 "마지막 내보내기: N월 N일" 형식으로 표시(없으면 문구 숨김)
- 로컬 저장 한계 안내 문구 고정 노출

### 7.6 친구 연결 — `#/connect` (PRD 기능: ⑤, 선택 기능)
- "초대 코드 만들기": 랜덤 코드(예: 6자리 영숫자) 생성 → `humanlog_invite_code`로 저장 → 클립보드 복사 버튼 제공
- "초대 코드 입력하기": 입력값을 `humanlog_connections` 배열에 저장(실제 서버 연동 없이, MVP 단계에서는 로컬에만 저장 — 아래 8절 한계 참고)
- 연결된 상대 목록 클릭 시 `#/connect/compare/{inviteId}`로 이동
- 하단에 "건너뛰어도 모든 핵심 기능을 사용할 수 있어요" 안내 고정 노출

### 7.7 인식 차이 비교 — `#/connect/compare/:inviteId` (PRD 기능: ⑤)
- 나의 `humanlog_responses`와 상대방 데이터(로컬 mock 또는 가져온 데이터) 중 **질문 ID가 겹치는 항목만** 좌/우 카드로 병렬 표시
- 답변이 동일할 때는 강조색을 넣지 않고 동일하게 표시(우열 강조 금지)
- 공통 질문이 없으면 Empty State 표시

### 7.8 알림 설정 — `#/settings/notifications` (PRD 기능: ⑥)
- `Notification.requestPermission()`으로 브라우저 알림 권한 요청
- 토글 On/Off, 시간대 선택(오전 9시 / 저녁 8시 / 밤 10시 중 택1 — 커스텀 시간 입력은 MVP 이후로 보류)
- 설정값은 `humanlog_notification_settings`에 저장
- 권한이 거부된 상태면 안내 문구로 대체하고 토글은 비활성화

### 7.9 설정 — `#/settings` (비기능 요구사항: 다크모드 등)
- 다크모드 토글: `humanlog_theme`에 저장하고 `<html data-theme="...">` 갱신
- 메뉴 리스트 클릭 시 각각 `#/settings/notifications`, `#/backup`, `#/connect`, `#/onboarding`으로 이동
- "기록 전체 초기화"는 확인 다이얼로그(`confirm()` 또는 커스텀 모달) 통과 후에만 `humanlog_responses` 삭제

## 8. 알려진 한계 (구현 시 코드 주석으로도 명시할 것)

- `localStorage` 기반이므로 기기 간 자동 동기화가 되지 않는다 → 사용자가 수동으로 `.xlsx` 백업/복원해야 한다.
- 친구 연결(⑤) 기능은 실제 서버가 없는 MVP 단계에서는 "같은 브라우저 내 mock 데이터 비교" 수준으로 구현하고,
  실제 다른 사람과의 실시간 비교가 필요해지면 별도 백엔드(DB) 도입이 전제되어야 한다는 점을 코드 주석과 `/connect` 화면 문구에 함께 남긴다.

## 9. 완료 기준 (PRD 6절 성공 기준 그대로 적용)

1. 홈에서 4지선다 답변 후 1초 이내 AI 관찰 문장이 표시되고, 같은 날 재방문 시 동일한 질문·답변·관찰이 유지된다.
2. `#/log`에서 최소 30일치 기록이 날짜별로 누락 없이 조회된다.
3. `#/backup`의 내보내기로 (날짜, 질문, 선택, AI 관찰) 4열의 `.xlsx`가 다운로드되고, 가져오기로 병합이 정상 반영된다.
4. Vercel 무료 배포 URL 접속 시 로그인 없이 데스크톱·모바일에서 레이아웃 깨짐 없이 핵심 기능이 동작한다.
5. 모든 화면 문구에서 MBTI식 라벨, 단정 표현, 외모·우열 비교가 발견되지 않는다.
6. 친구 연결·알림 기능을 한 번도 사용하지 않아도 홈 → 기록 → 백업 흐름이 완결된다.

## 10. 개발 순서 제안 (AI 코딩 도구에 단계별로 지시할 때)

1. 폴더 구조 및 `tokens.css` / `reset.css` 생성
2. `router.js` + `app.js`로 해시 라우팅 셸 구성 (빈 화면이라도 전환되는지 먼저 확인)
3. `questions.js`, `insights.js`, `storage.js` mock 데이터/유틸 작성
4. 홈 화면(`views/home.js`) 완성 — 이 프로젝트의 핵심이므로 가장 먼저 완성도 있게 구현
5. 기록 아카이브 + 기록 상세
6. 데이터 백업(xlsx 연동)
7. 설정 + 알림 설정 + 다크모드
8. 온보딩
9. 친구 연결 + 인식 차이 비교 (선택 기능이므로 가장 마지막)
10. `vercel.json` 점검 후 Vercel 배포
